Desktop

NAME

Desktop

DESCRIPTION

Synopsis

The desktop's look and feel is inspired by desktops of the 1990s, such as:

Windows 95 Explorer

GNOME v1

K Desktop Enviroment v1

Window Control

On the top right corner of each window, there are three buttons:

Minimise (-), Maximise ([]), Close(x)

Minimise hides the window to the taskbar (see Taskbar)

Maximise, in the case of the three window buttons, makes the window fullscreen

Maximise, in the case of the Taskbar, restores the window to the desktop

Close simply closes the application

Drag a window to a different place on the desktop by clicking and dragging the window header to its desired location

Resize a window manually by clicking and dragging the tab on the bottom right corner of the window

Taskbar

The taskbar is the block of colour that sits at the bottom of your screen

Click on the buttons to maximise (see Window Control) your open apps

Click on the "Start" button to access your Start Menu

Start Menu

The Start Menu is a menu that contains buttons to open your installed programs

It contains submenus such as "Installed Programs" and "Utilities", which can be navigated by hovering your cursor over them

To close it, simple click on the Start button again

Wallpaper

Change your wallpaper to a different solid color or a custom image in Settings > Wallpaper (see Applications)

Filesystem

NAME

Filesystem

DESCRIPTION

Synopsis

sigmaOS's filesystem is different to a traditional filesystem, as instead of a physical location on a user's hard drive being written and read to,

sigmaOS simply uses JSON to store files and folders in the user's browser's localStorage cache

A basic sigmaOS filesystem typically looks like:

{"root":{"folder":"subfolder"}}

As JSON has a {"key":"value"} syntax

Usage

Users can interact with the filesystem by using the shell, Notes, or File Explorer (all see Applications)

Where you are in the filesystem is determined by the file path

A typical filepath looks like:

root/home/Documents/coolstory.txt

The top of the filesystem is the path root/

Permissions

NAME

Permissions

DESCRIPTION

Synopsis

All users of sigmaOS have the absolute highest privilege possible; all files and folders are modifiable and removable, even if vital to the system

sigmaOS repairs itself if any critical files are missing, however

Explorer

NAME

Explorer

DESCRIPTION

Synopsis

Graphical File Explorer

On the left, shows the users path (see filesystem) where you can click on any part of it to go up towards the top of the filesystem

On the right are all the folders and files at that path

A folder is indicated by a yellow square, whereas a file is indicated by a blue square

Click on a file or folder to select it, double click on a folder to go into it

On the top, there is a ribbon bar to delete any selected items, create a new file or folder, or to refresh for any new files or folders modified by other programs

Notes

NAME

Notes

DESCRIPTION

Synopsis

Graphical Text Editor

Type or edit in the text box, when you are done, click the Save button. A minimalistic file picker will open, letting you save anywhere

(This can be buggy, and will default to your home folder (see filesystem) if it cannot find where the original file/path was/is)

Or to edit an existing file, click the Open File button and the same file picker will appear, letting you edit your documents

Shell

NAME

Shell

DESCRIPTION

Synopsis

A shell is a direct way to talk to the core (kernel) of your operating system

Commands

pwd - Print the current directory you are in

ls - List all folders and files in your current directory

cd (name) - change your current directory to an existing directory; to go up a directory, type cd ..

mkdir (name) - Make a directory

rmdir (name) - Remove a directory and all subdirectories within it

touch (name) - Create an empty file

lsblk - list all the "partitions" that sigmaOS is installed on (cosmetic purposes only)

clear - Clear the terminal

help - Print a list of commands whenever you need it

(By default, sigmaOS comes with: neofetch - Prints some system information)
